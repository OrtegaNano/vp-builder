# 03 — Doble Promesa

Dos pilares complementarios. Ninguno funciona solo. Juntos, cada stakeholder tiene razón para decir sí.

---

## Discovery

1. Si tuvieras que convencer a dos personas opuestas (CEO vs director de riesgos), ¿qué le dirías a cada uno?
2. ¿Cuál es el beneficio "sexy"? (lo que emociona al decisor)
3. ¿Cuál es el beneficio "defensivo"? (lo que neutraliza al bloqueador)
4. ¿Quiénes son los stakeholders típicos en la decisión de compra?

---

## Prompt de generación

```
CONTEXTO: {intake + discovery + secciones anteriores}

TAREA: Genera "Doble Promesa".

ESTRUCTURA:

1. Identifica las dos promesas (del discovery o inferidas del producto):
   - Promesa A = el beneficio "sexy" (velocidad, productividad, ventaja competitiva, innovación)
   - Promesa B = el beneficio "defensivo" (seguridad, compliance, control, calidad, reducción de riesgo)

2. PROMESA A SOLA NO BASTA:
   "Si solo prometemos [A], el [rol bloqueador] mata el deal porque pregunta [objeción concreta]."
   Ejemplo de conversación real de ventas.

3. PROMESA B SOLA NO BASTA:
   "Si solo prometemos [B], el [decisor] pierde interés porque no ve [beneficio que busca]."
   Ejemplo de conversación real.

4. JUNTAS HACEN LA VENTA FÁCIL:
   Para cada stakeholder del usuario:
   - El [ROL] obtiene [qué exactamente]
   (4-6 stakeholders típicamente)
   
   Cierre: "No le estamos pidiendo a una persona que defienda la compra contra resistencia interna — le estamos dando a cada stakeholder una razón para decir sí."

REGLAS:
- Stakeholders SOLO del usuario/research.
- Los ejemplos de bloqueo deben sonar como conversaciones reales de venta en ese sector.
- Las dos promesas deben ser GENUINAMENTE complementarias, no dos formas de decir lo mismo.

FRAMEWORK: Dual value proposition (eficiencia + protección). Referencia: la estructura "gain creators + pain relievers" del Value Proposition Canvas de Strategyzer.
```
