# 04 — Messaging por Audiencia

Misma propuesta, diferente ángulo. Cada decisor escucha exactamente lo que necesita oír.

---

## Discovery

1. Para cada stakeholder: ¿cuál es su mayor dolor? ¿Qué KPI le quita el sueño?
2. ¿Qué objeciones te pone cada uno?
3. ¿Quién es campeón interno vs. firmador?
4. Los usuarios finales: ¿frustración, curiosidad, miedo, indiferencia?

---

## Prompt de generación

```
CONTEXTO: {intake + discovery + secciones anteriores}

TAREA: Genera "Messaging por Audiencia".

Para CADA stakeholder identificado:

### Para [Rol / Título]

2-3 párrafos que:
- ABREN con el dolor específico de ESA persona (no el dolor genérico de la empresa)
- CONECTAN con lo que el producto resuelve PARA ELLA concretamente
- NEUTRALIZAN la objeción típica de ese rol
- Si hay ROI o métricas de secciones anteriores, las incluyen donde sean relevantes para ESE stakeholder
- CIERRAN con urgencia o coste de oportunidad personalizado

TONO por audiencia:
- C-suite: estratégico, competitivo, board-ready. "Tus competidores están..."
- Managers / L&D: práctico, medible. "Puedes demostrar a liderazgo que..."
- Compliance / Riesgo: defensivo, regulatorio. "Tu mayor riesgo ahora mismo es..."
- Usuarios finales: empático, concreto. "Sabes ese task que te come 3 horas..."

Para C-suite, incluir un bloque de ROI:
"A [precio], si el producto ahorra [X horas/semana] a un coste cargado de [Y/hora], son [Z] en productividad recuperada por año. La inversión se paga en [N meses]."

REGLAS:
- No incluyas audiencias que el usuario no mencionó.
- Cada pitch debe sonar como conversación cara a cara, no párrafo de web.
- La voz cambia por audiencia pero el mensaje central es consistente.

FRAMEWORK: Buyer persona messaging matrix. Cada stakeholder tiene un dolor, una objeción, un trigger y un resultado deseado diferentes.
```
